## 15. 작성 팁

- **문제-접근-결과** 구조를 유지한다.
- “열심히 했다”보다 **무엇을 판단했고 어떤 근거로 결정했는지**를 쓴다.
- 가능하면 결과는 숫자로 표현한다.
- 영어 기술 용어는 그대로 쓰고, 설명은 한국어로 정리한다.
- 팀 프로젝트는 반드시 **내 기여도**를 분리해서 적는다.
- 후공정 직무라면 공정명, failure mode, 데이터 분석 방식, 검증 방법을 꼭 넣는다.
- 내용이 길어질 경우 프로젝트당 1~2페이지 내로 정리하고, 상세 내용은 별첨으로 분리한다.
